## 👋 Simple CMS REST API Project
(주)맑은기술 백엔드 과제로 진행된 Simple CMS REST API 프로젝트입니다. 

기본적인 요구사항인 콘텐츠 CRUD 및 권한 제어를 구현하였으며, 추가로 QueryDSL을 활용한 동적 검색과 계층형 댓글 시스템을 포함하고 있습니다.
##
### 프로젝트 실행 방법
**1. 환경 요구사항**
- Java 25 이상
- Gradle 8.x 이상
- H2 Database (별도 설치 불필요, In-memory 사용)

<br>

**2. 프로젝트 빌드 및 실행**

터미널(또는 CMD)에서 프로젝트 루트 디렉토리로 이동한 후 아래 명령어를 입력합니다.
``` {bash}
1. 레포지토리 클론 (또는 압축 해제 후 해당 폴더 이동)
git clone https://github.com/tickling1/malgn-assignment.git
cd malgn-assignment

2. 빌드 및 실행 (Unix/Linux/macOS)
./gradlew bootRun

3. 빌드 및 실행 (Windows)
./gradlew.bat bootRun
```
<br>

- H2 Console (DB 확인): http://localhost:8080/h2-console

### 구현 내용 및 추가 구현 기능
**보안 및 권한 검증 - Spring Session JDBC를 선택(기본 요구사항)**

**1. 운영 영속성**

CMS 관리자는 긴 본문의 공지사항을 작성하거나 시스템의 복잡한 정책을 설정하는 등 호흡이 긴 업무를 수행하는 경우가 많다고 생각합니다. 만약 서버의 기본 인메모리 세션 방식을 사용한다면, 새로운 기능 배포나 서버 점검을 위해 어플리케이션을 재시작하는 순간 모든 관리자의 세션이 끊겨 작업 중인 데이터가 유실되거나 강제 로그아웃되는 문제 상황이 발생합니다. Spring Session JDBC를 도입함으로써 세션 정보는 어플리케이션의 메모리가 아닌 외부 데이터베이스(`SPRING_SESSION`)에 영구적으로 저장하도록하여 애플리케이션의 생명주기와 세션의 생명주기를 분리했습니다. 이를 통해 서버 점검이나 배포 중에도 관리자의 인증 상태를 유지하여 중단 없는 운영 환경이 가능해집니다.

**2. 실시간 보안 통제권**

또한, 관리자는 시스템 내부에서 발생하는 모든 보안 상황을 통제할 수 있어야 된다고 생각합니다. JDBC 기반의 세션 방식은 관리자에게 실시간 제어권을 부여합니다. 관리자는 데이터베이스를 통해 현재 접속 중인 사용자 수와 활동 중인 세션을 실시간으로 모니터링할 수 있습니다. 예를 들어, 특정 사용자의 계정이 도용되거나 보안 정책을 위반하는 행위가 감지될 경우, 관리자는 DB에 저장된 세션 레코드를 삭제함으로써 즉각적으로 해당 유저를 강제 로그아웃시키는 제재 조치를 취할 수 있습니다. 

일반적인 JWT 방식은 Refresh 토큰을 도입하더라도, 이미 발행된 Access 토큰에 대한 즉각적인 무효화가 불가능합니다. 서버는 토큰 자체의 유효성만 검증할 뿐 매 요청마다 상태를 확인하지 않기 때문에, 토큰이 탈취되거나 보안 정책 위반이 감지되어도 해당 토큰이 만료될 때까지 서버는 제어권을 상실하는 사각지대가 발생합니다. CMS 관리자에게 필요한 것은 "5분 뒤에 만료되는 토큰"이 아니라, "지금 당장 차단되는 세션" 입니다. 

물론 JWT도 블랙리스트를 추가 구현하여 제어할 순 있으나, 이는 무상태(Stateless)라는 JWT의 이점을 희생시키고 시스템 복잡도를 높이는 결과라고 생각했습니다. 즉, 서버가 매 요청마다 "이 토큰이 차단되었나?"를 확인하기 위해 Redis나 DB를 조회하게 된다면, 그것은 이미 무상태가 아닌 상태 유지 시스템이 된 것이라고 생각합니다. 이런 이유들로 본 프로젝트는 JWT를 쓰고 억지로 상태를 관리하는 방식 대신, 처음부터 세션을 외부 DB에 저장하는 직관적인 설계를 선택했습니다.

**3. 향후 기능 계획**

마지막으로 향후 기능을 더 만들고 싶다면 사용자가 현재 로그인된 모든 기기(PC, 모바일 등)의 리스트를 확인하고, 보안이 우려되는 특정 기기나 공공 장소의 세션을 원격으로 즉시 종료할 수 있는 기능이나 사용자의 접속 환경(IP 주소, 브라우저 정보 등)이 세션 생성 시점과 급격하게 달라질 경우, 이를 실시간으로 감지하여 보안을 강화하는 기능을 구현해보고 싶습니다.

- Spring Security를 활용하여 USER, ADMIN 권한을 구분했습니다.
- 콘텐츠 수정 및 삭제 시, 작성자 본인 여부를 검증하는 로직을 서비스 레이어에 구현했습니다.
- 관리자(ADMIN) 권한을 가진 사용자는 모든 콘텐츠에 대해 관리 권한(수정,삭제)을 가질 수 있도록 예외 로직을 적용했습니다.

**4. 계층형 댓글 시스템 (추가 구현)**

<img width="933" height="1000" alt="image" src="https://github.com/user-attachments/assets/053bdb30-a278-4f35-b127-c336d0cb2bf7" />

- Comments(댓글) 엔티티 내에서 자기 자신을 참조하는 parentComment 필드를 설계하여 무한 계층 구조의 기반을 마련했습니다.
- 부모 댓글 아래에 자식 댓글(답글)이 리스트 형태로 포함되는 계층형 구조입니다.
- Jackson 어노테이션(@JsonPropertyOrder, @JsonInclude)을 활용하여 API 응답 시 게시글은 최신순(내림차순)으로 보여주고,댓글과 대댓글은 대화의 맥락을 유지하기 위해 작성 순서(오름차순)로 배치되도록 하고, 불필요한 필드(답글이 비어있는 경우)는 노출되지 않도록 조정했습니다.
